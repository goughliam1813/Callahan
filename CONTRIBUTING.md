# Contributing to Callahan CI

Thank you for helping make Callahan better! 🛸

## Getting Started

1. Fork the repo and clone your fork
2. Create a feature branch: `git checkout -b feature/my-thing`
3. Make your changes — see development setup in [README.md](README.md#️-development)
4. Run tests: `cd backend && go test ./...`
5. Open a PR with a clear description

## Code Style

- **Go**: `gofmt`, keep functions small, add comments on exported types
- **TypeScript**: Prettier defaults, typed props, no `any` unless unavoidable
- **Commits**: Use conventional commits — `feat:`, `fix:`, `docs:`, `chore:`

## Reporting Bugs

Open an issue with: OS, Go/Node versions, steps to reproduce, and the error output.

## Feature Requests

Open a discussion first for larger features so we can align before you invest the time.
